1.什么是面向对象（oop）

  • 面对过程

    • 步骤清晰
    • 适合处理一些简单的问题

  • 面对对象

    • 物以类聚，分类的思想，思考解决问题需要哪些分类，然后对这些分类进行单独思考，最后对每一个类进行面对对象思考
    • 适合处理一些复杂的问题

  • 本质：以类的方法组织代码，以对象的形式封装数据

  • 特征：封装，继承，多态

  • 从认识角度，先有对象后抽象出类

  • 从代码角度，现有类后创建对象

2.方法回顾

  • 方法的调用

    • 静态方法：static修饰，可以直接使用
    • 非静态方法：需要先实例化类对象类型 对象名称 = new 对象类型()
    • 静态方法中不能直接调用非静态方法

  • 值传递和引用传递

    • 值传递
    • 引用传递(对象)

3.创建与初始化对象

  • 类里面只有属性和方法

  • this表示当前类

  • 构造器

    • 使用new必须有构造器
    • 不写会有默认构造器（隐式定义）
    • 必须和类的名字相同
    • 没有返回值
    • IDEA中使用alt+insert自动生成构造器

4.三大特征

封装

  • 该露的露，该藏的藏

  • 高内聚低耦合，尽量暴露少量方法给外部使用

  • 属性私有，get/set

    • private修饰属性

    • 通过get() / set()方法获取操作属性，同时可以进行安全性检查

    • IDEAalt+insert -> getter setter可以直接生成get/set方法

  • 意义

    1. 提高程序安全性，保护数据
    2. 隐藏代码的实现细节
    3. 统一接口
    4. 提高系统可维护性

继承

  • 继承的本质是对某一批类的抽象，从而实现对现实世界的更好的建模
  • 通过extends继承，只能单继承
  • 子类和父类之间应该具有’is a’的关系，比如：Student is a Person
  • 子类具有父类的所有public方法和属性，私有（private）无法被继承
  • 所有类都默认继承Object类

super - this

  • this：调用当前类
  • super：调用父类
  • 子类的构造器中默认有父类的无参构造器，如果在子类构造器中显示写出父类的构造器必须在第一句

方法重写

  • 需要有继承关系，子类重写父类的方法。
  • 方法名必须相同
  • 参数列表必须相同
  • 必须是非静态方法
  • 修 饰符：范围可以扩大不能缩小：public > protected> default>private
  • 抛出异常：范围可以被缩小不能扩大

多态

  • 同一方法可以根据发送对象的不同而采用不同的行为方式
  • 注意事项
    1. 多态是方法的多态，
    2. 父类和子类，有联系 才类型转化
    3. 存在条件： 继承关系，方法重写，父类的引用指向子类对象Father f1 = new Son()
    4. 是否能调用方法看引用类型
      1. 上例中Father是引用类型Son是实际类型
      2. 引用类型有，走引用类型
      3. 引用类型有，且实际类型中被重写，走实际类型
      4. 引用类型没有，实际类型有，需要类型转换
  • instanceof + 类型转换
    • X instanceof Y 判断XY之间是否存在父子关系（只要是一条线上的都返回True，Ex：Object > Father > Son）
    • (class) variableName 强制转换

5.抽象类

  • abstract修饰
  • 抽象类的所有方法需要子类去重写
  • 特点
    • 作为约束
    • 不能实例化(不能new)
    • 抽象类可以写普通方法/抽象方法
    • 抽象方法必须写在抽象类中

6.接口

  • 只有规范，没有自己的实现，不能写方法，不能被实例化
  • 规范和实现分离
  • 可以实现多继承
  • 通过类来实现接口,必须重写方法，通过implement关键字实现
  • 通过implement可以实现多个接口

7.内部类

成员内部类

  • 内部类会生成独立的文件
  • 内部类可以直接获取到外部类的私有属性
  • 内部类属性和外部类属性名相同，优先获取内部的属性，获取外部类则需要 Outer.this.name
  • 内部类不能包含静态属性，但可以包含静态常量

静态内部类

  • j级别上约等于外部类。
  • 可以直接创建，不需要先创建外部类对象。
  • 可以包括静态属性。
  • 静态内部类中的方法调用外部类属性需要先创建外部类对象。

局部内部类

  • 定义在外部类的方法里面
  • 不能添加修饰符如：private
  • 可以直接访问外部类的属性
  • 不能包含静态属性

匿名内部类

  • 类似于局部内部类
  • 必须继承一个父类或者实现一个接口
  • 定义类，实现类，创建对象的语法合并
  • 只能创建一个对象
